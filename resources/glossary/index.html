Glossary

The language of AI-driven campaign automation.

These are the terms we use consistently across every engagement. If you see a term on this site, its exact definition is here.

Core concepts

The foundational terms.

Core concept

Guardrail-driven automation

An AI automation approach where every agent action is constrained by explicit, auditable rules. The guardrail layer defines what the agent can and cannot do — preventing runaway spend, brand violations, or unapproved actions. The rules are visible, editable, and logged.

Core concept

Trigger/action/impact

The accountability schema for every automated action. Trigger: the condition that caused the agent to act. Action: what the agent did. Impact: the measured outcome with a timestamp. No action is logged without all three fields.

Framework

Agentic marketing

Marketing operations run by AI agents that can plan, execute, and adjust campaigns within defined constraints. Not just automation — agents make decisions, not just follow scripts.

Framework

TAI schema

Shorthand for trigger/action/impact. The three-field log entry format used in all campaign automation activity logs. See: /results/trigger-action-impact

Products

What we deliver.

Product

Campaign Automation Audit

A human-led diagnostic, and the first sprint of every engagement, that analyses your campaign data via read-only access to your key accounts (Google Ads, GA4, Search Console) to identify PPC waste, SEO/PPC gaps, and AI automation opportunities. Produces a 30-day action plan, and is how new clients first work with our team.

Product

Automation Sprint

A 2–4 week engagement ($5,000–$15,000) that builds one specific campaign automation workflow. Delivered with guardrail config, prompt packs, AGENTS.md, and a training session. The client owns the result.

Product

Campaign Strategy Portal

A recurring subscription ($500–$2,500/month) that provides monthly AI-generated strategy, recommendations, assets, and agent instructions for your campaigns.

Technical terms

How the systems work.

Technical

AGENTS.md

A markdown file that defines the rules, constraints, personas, and workflows for AI agents operating on a campaign. The guardrail configuration document. Every sprint delivers one.

Technical

Prompt pack

A structured set of reusable prompts for a specific campaign task — PPC brief generation, SEO keyword research, ad copy creation, or lead scoring. Delivered as part of every sprint.

Technical

Guardrail

A rule that constrains AI agent behaviour. Examples: daily spend cap, brand exclusion list, maximum bid change per cycle, required human approval for actions above a threshold.

Metrics

How we measure impact.

Metric

CPA (Cost Per Acquisition)

The total cost to acquire one customer or conversion. A core KPI for PPC campaigns. CPA reduction is the primary metric for the PPC Intelligence Sprint.

Metric

LTV (Lifetime Value)

The predicted total revenue from a customer over their relationship with the business. Used in bid strategy and budget allocation decisions.

Expanded glossary

The operator's vocabulary.

The terms that matter most for AI marketing automation, ordered by relevance — each tagged as a platform you adopt, a workflow you run, or a process you practise.

Process

Bounded autonomy

An operating model where an AI agent acts freely but only within explicitly defined limits, instead of asking permission for every move. The core of guardrail-driven automation.

Workflow

Human-in-the-loop

Requiring human approval at defined points before an agent's action takes effect.

Workflow

Human-on-the-loop

Letting an agent act autonomously while a human monitors and can intervene, rather than approving each step.

Process

Blast radius

The maximum scope of damage an autonomous action could cause if it goes wrong — the variable you cap before granting autonomy.

Process

Reversal token

A captured handle that lets a system undo an agent's action, making otherwise risky autonomy acceptable because it is reversible.

Workflow

Dry-run mode

Executing an agent's planned actions without committing them, so a human can review intended changes first.

Platform

Agent

A system where an AI decides its own sequence of actions and tool calls to reach a goal, versus a fixed deterministic workflow.

Workflow

Agentic workflow

A process that mixes deterministic steps with points where an agent reasons, chooses tools, or handles ambiguity.

Platform

MCP (Model Context Protocol)

An open standard for connecting AI models to external tools and data through a uniform server interface.

Platform

Tool use

A model invoking external functions or APIs, with the host executing the call and returning results into context.

Process

Lead scoring

Ranking leads by likelihood to convert using fit and behaviour signals, so sales works the warmest first.

Process

Predictive lead scoring

Lead scoring where a model, not hand-tuned rules, learns which attributes and actions predict conversion from historical data.

Workflow

Journey orchestration

Coordinating a contact's experience across channels in response to behaviour, rather than running channels as silos.

Platform

Smart Bidding

Google's auction-time bidding that uses signals and machine learning to optimise toward a target cost or value goal.

Platform

Performance Max

Google's goal-based campaign type that auto-allocates across all its inventory from one asset group and conversion goals.

Platform

Advantage+

Meta's automation suite that hands targeting, placement, and budget decisions to its models given a conversion objective.

Process

Autonomous bidding

Optimisation where the platform sets bids and budgets with minimal human input, contrasted with rules-based control.

Process

Budget pacing

Controlling spend velocity across a flight so a budget is not exhausted early or left unspent.

Process

tROAS

Target return on ad spend — a bid strategy that sets bids to hit a desired revenue-to-cost ratio.

Process

Incrementality testing

Measuring the conversions an ad actually caused versus those that would have happened anyway, via controlled holdouts.

Process

Multi-touch attribution

Distributing conversion credit across the several touchpoints in a journey rather than crediting only one.

Process

Marketing mix modeling

A top-down statistical model attributing outcomes to channels using aggregate spend and results, resilient to signal loss.

Process

RevOps

Revenue operations — unifying marketing, sales, and success operations under shared data, process, and accountability for revenue.

Process

ICP

Ideal customer profile — the firmographic and behavioural definition of the accounts worth pursuing.

Process

Intent data

Signals that an account is researching a category — first-party from your properties, third-party from external networks.

Process

Win probability

A model-derived likelihood that an open deal closes, used to prioritise effort and sharpen forecasts.

Process

Speed-to-lead

The elapsed time from inbound interest to first sales contact, where minutes materially change conversion odds.

Platform

CDP (Customer Data Platform)

Software that unifies customer data from many sources into persistent profiles for activation.

Process

Identity resolution

Stitching disparate identifiers into one persistent profile so a customer is recognised across devices and channels.

Workflow

Reverse ETL

Pushing modeled data from the warehouse back into operational tools like CRMs and ad platforms for activation.

Platform

iPaaS

Integration platform as a service — hosted middleware for connecting apps and syncing data without custom infrastructure.

Platform

Webhook

An HTTP callback fired by an event so downstream systems react in real time instead of polling for changes.

Platform

RPA

Robotic process automation — software bots that automate UI-driven, repetitive tasks across systems lacking APIs.

Workflow

Self-healing automation

An automation that adapts when an underlying interface or schema changes instead of breaking and needing manual repair.

Process

Send-time optimization

Choosing each recipient's individual best send moment from their past engagement to lift opens and clicks.

Process

Sender reputation

A mailbox provider's trust score for your sending identity — the dominant factor in whether mail reaches the inbox.

Platform

DMARC

An email-authentication policy tying SPF and DKIM to the visible From domain and telling receivers what to do with failures.

Process

List hygiene

Routinely validating and pruning addresses to remove bounces, traps, and dead contacts that drag reputation down.

Process

AEO (Answer Engine Optimization)

Structuring content so AI systems extract and surface it as a direct answer.

Process

GEO (Generative Engine Optimization)

Shaping content and presence so LLMs cite your brand as a source in synthesized answers.

Platform

AI Overviews

Google's generative summaries atop search results that answer directly and depress click-through to ranked links.

Process

AI governance

The policies, controls, and accountability structures for deploying AI responsibly and within legal and brand limits.

Platform

Trust layer

A wrapper around model calls enforcing grounding, data masking, retention rules, and content checks before output is used.

Process

Least-privilege access

Granting a system or agent only the minimum permissions its task requires, capping the damage of misuse or compromise.

Process

Prompt injection

An attack that smuggles instructions into model input to override intended behaviour — a key risk when agents read external content.

Platform

RAG (Retrieval-Augmented Generation)

Fetching relevant documents at query time and grounding the model's answer in them, so claims trace to a source.
