Rahul Verma

Marketing Data Engineer
San Jose, CA · he/him · 8 yrs · Data & analytics

Warehouse-modeled data, synced to ad and CRM tools with tests gating it

This is an illustrative profile of a role we’re actively filling — a representative archetype, not a current team member. If it describes you, apply for this role.

The profile

Rahul Verma is a Marketing Data Engineer in San Jose with eight years spent on the layer most automation skips: the pipeline. He starts in the warehouse, modeling audience and conversion data in dbt on BigQuery so that what flows downstream is defined once, documented, and reproducible rather than stitched together across a dozen tool UIs.

His rule is simple: nothing syncs until the data earns it. dbt tests run on every model, and a failing test stops the push before bad segments or stale revenue numbers reach the ad platforms or CRM. That guardrail matters because reverse ETL hands automation real money decisions, and a silent null or a duplicated row becomes wasted spend or a misfired campaign.

Rahul builds those guarantees with Hightouch handling activation, so marketing teams sync confidently and keep an audit trail of exactly what moved, when, and why.

What this builder owns

dbt-tested audience models on BigQuery

Defines segments as version-controlled dbt models in BigQuery with schema and freshness tests that must pass before any audience is eligible to sync.

Test-gated reverse ETL with Hightouch

Wires Hightouch syncs to dbt test status so a failed assertion blocks the push to ad and CRM tools instead of shipping bad rows downstream.

Auditable warehouse-to-CRM sync logs

Instruments every Hightouch sync with run-level logging that records which records changed, the model version behind them, and the trigger.

Reversible activation with staged rollback

Builds reverse ETL flows that snapshot prior synced state so a faulty audience push can be rolled back to the last known-good model run.
