Yuki Tanaka

Analytics & Measurement Engineer
Victoria, BC · they/them · 10 yrs · Data & analytics

If it isn't instrumented, it doesn't get automated

This is an illustrative profile of a role we’re actively filling — a representative archetype, not a current team member. If it describes you, apply for this role.

The profile

Yuki Tanaka builds the measurement layer that automation actually runs on. Over ten years they have specialized in the foundation most teams rush past: server-side GTM containers, clean event schemas, and GA4 setups that report the same number twice in a row. Based in Victoria, BC, they treat instrumentation as the precondition for any automated decision, not an afterthought.

The work starts with definitions. Yuki maps every event, parameter, and conversion before a single tag fires, then pipes raw streams into BigQuery so attribution can be recomputed and audited rather than taken on faith. When numbers are questioned in a board meeting, the lineage is already there to defend them.

That discipline is exactly why guardrails matter here. An automation that optimizes toward a broken conversion event will spend budget confidently in the wrong direction. Yuki refuses to let a system act on data it cannot trace, so every signal feeding an automated rule is validated, versioned, and reversible first.

What this builder owns

Server-side GTM container with consent gating

Stands up a server-side GTM setup that strips PII, enforces consent state, and forwards clean events to GA4 and ad platforms with full request logging.

Versioned event schema and naming contract

Defines a documented event and parameter taxonomy enforced at the tag layer, so every team logs the same action the same way and schema changes are reviewed before they ship.

BigQuery attribution model you can audit

Exports raw GA4 event data into BigQuery and rebuilds attribution as queryable SQL, so any reported number can be traced back to source touchpoints and recomputed on demand.

Automation-readiness data quality checks

Adds scheduled validation in BigQuery that flags missing conversions, duplicate events, and tracking drift before any number is allowed to drive an automated bidding or budget rule.
