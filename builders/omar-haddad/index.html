Omar Haddad

Microsoft Copilot Architect
Remote — Montréal, QC · he/him · 9 yrs · AI agents & MCP

Copilot agents your IT department actually approves

This is an illustrative profile of a role we’re actively filling — a representative archetype, not a current team member. If it describes you, apply for this role.

The profile

Omar Haddad has spent nine years in the Microsoft stack, the last several building Copilot agents that live where enterprise IT can see them. Working remotely from Montréal, he sits at the seam between marketing teams who want autonomy and security teams who answer for every data flow, and he speaks both languages fluently.

His specialty is AI agents and MCP wired into real systems: Copilot Studio agents, Power Automate flows, and Dataverse, all grounded in Azure data. The guardrails are not an afterthought. Role-based access, DLP policies, and connector boundaries are designed in from the first sketch, so an agent can only touch the campaign data and actions it has been granted, and every step is logged.

That discipline is why his work clears review. In a Copilot deployment, the difference between a pilot and a production rollout is usually whether IT trusts the permission model. Omar builds so that they do, with approval gates and reversible actions baked into each flow.

What this builder owns

Governed Copilot Studio marketing agent

A Copilot Studio agent that answers campaign questions and drafts actions against Dataverse, scoped by role-based access so it only reads and writes what each user is permitted to.

Approval-gated Power Automate campaign flows

Power Automate flows that route content launches and budget changes through human approval thresholds before anything publishes, with full run history retained.

Azure-grounded, DLP-compliant data connectors

Connector layers that ground Copilot agents in Azure marketing data while DLP policies block any path that would move regulated fields outside approved boundaries.

Reversible MCP action toolkit

MCP-based agent tools where each marketing action is bounded, logged in Dataverse, and reversible, so a wrong call can be traced and rolled back cleanly.
